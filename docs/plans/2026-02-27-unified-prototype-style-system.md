# SkillsIndex 统一风格设计规范（参考站点融合版）

日期：2026-02-27
适用范围：SkillsIndex 全站原型（首页、列表、详情、管理端）
参考站点：
- https://skillsmp.com/zh
- https://mcpmarket.com/zh/tools/skills
- https://lobehub.com/zh/skills

## 1. 目标与边界

### 1.1 目标
1. 形成一套跨页面一致的视觉语言，避免“每页一个风格”。
2. 首页以 Skill 卡片浏览与搜索为核心，减少噪音入口。
3. 管理端保持专业、清晰、可扫描，不牺牲密度。
4. 暗黑与亮色主题保持同构层级，不出现“亮色看不清、暗色没边界”。

### 1.2 边界
1. 本规范优先约束视觉层与交互层级，不改变业务信息架构。
2. 不引入大面积花哨色块，不走拟物重装饰风格。

## 2. 参考站点抽象（必须吸收）

### 2.1 共性特征
1. 搜索优先：首屏先给搜索，再给筛选，再给卡片。
2. 卡片主导：内容主体是高密度卡片网格，而不是说明面板。
3. 蓝系克制：品牌蓝用于强调与选中，不作为大面积噪音背景。
4. 信息层级明确：标题、元信息、标签、动作层级稳定。
5. 可读性优先：边界明确、对比稳定、行距克制。

### 2.2 禁止项
1. 首页出现与 Skill 浏览无关的管理入口干扰主体注意力。
2. 未选中卡片在暗黑主题下“无边界、融背景”。
3. 用高饱和色块充当普通信息容器。

## 3. 统一视觉系统

### 3.1 颜色策略

#### 首页（推荐）
- Dark Root: `#0B1326`
- Dark Card: `#1B2E57`
- Dark Neutral Border: `#5479B3`
- Dark Selected Border: `#BBD4FF`
- Light Root: `#F8FAFC`
- Light Card: `#FFFFFF`
- Light Soft Card: `#EEF3FB`
- Primary Accent: `#2563EB`

#### 管理端（推荐）
- Dark Root: `#0B1326`
- Dark Header: `#12213F`
- Dark Primary Card: `#1B2E57`
- Dark Context Card: `#1F3B62`
- Light Header Soft: `#E2EAF6`
- Light Primary Card: `#FFFFFF`
- Light Secondary Card: `#F8FAFC`

#### 语义色（仅语义用途）
- Success: `#16A34A`
- Warning: `#D97706`
- Error: `#DC2626`

### 3.2 字体与层级
1. 主字体：`Noto Sans SC`
2. 技术/日志：`JetBrains Mono`
3. 字号层级：`31 / 24 / 16 / 14 / 12 / 11`
4. 标题与正文对比必须明显，正文不低于可读阈值。

### 3.3 间距与边框
1. 卡片圆角基准：`14-18`
2. 卡片边框：
- 暗黑未选中：`1-2px`，必须可见
- 暗黑选中：比未选中更亮
- 亮色：`1px` 低对比边框
3. 栅格间距：`12/14/16` 统一节奏。

## 4. 首页模板规范（Search-first）

### 4.1 首屏结构顺序（必须）
1. 顶部导航
2. 搜索条（主搜索 + 附加条件）
3. 筛选 Chips
4. Skill 卡片网格（主体）

### 4.2 卡片网格规范
1. 桌面：高密度网格优先（2~4 列，按画布宽度调整）。
2. 卡片内容：标题 > 一行描述 > 标签/评分 > 元数据。
3. 选中卡：背景或边框强化，保持克制。
4. 未选中卡：必须清楚看出边界与分割。

### 4.3 文案密度
1. 每张卡尽量两行内完成主要信息表达。
2. 不在卡片堆叠长段说明。

## 5. 管理端模板规范（Primary + Context）

### 5.1 结构比例
1. 主工作区 > 右侧上下文区（视觉权重始终主区更高）。
2. 右侧动作区强调“可操作性”，非操作信息降噪。

### 5.2 页面族统一
以下页面族必须共享同一视觉语法：
1. Records Governance / Sync Policy / Sync Ledger / Job Orchestration
2. 暗色与亮色版本在层级上保持镜像关系。

## 6. 交互与可用性约束

1. 所有可点击元素必须有 `cursor` 与 hover/focus 反馈。
2. 关键操作（危险动作）必须与普通动作显著区分。
3. 不依赖颜色单独传达状态，需配合文字/图标。

## 7. 验收标准（执行级）

### 7.1 首页
1. 用户 1 秒内定位搜索区域。
2. 用户 3 秒内理解筛选状态与卡片选中状态。
3. 未选中卡片在暗黑模式下边界可见。

### 7.2 管理端
1. 主区与右区视觉优先级正确。
2. 同步/治理页面族风格一致。
3. 不出现大面积高饱和噪音块。

### 7.3 全局
1. 暗黑与亮色主题均可读。
2. 不新增结构裁切/遮挡问题。

## 8. 实施顺序（建议）

1. 第一阶段：首页（dark + light）统一 Search-first。
2. 第二阶段：治理/同步页面族风格统一。
3. 第三阶段：全站边框与状态体系一致化。
4. 第四阶段：逐页验收打分与回归。

## 9. 后续输出物

1. 页面级评分表（按本规范逐页打分）。
2. 风格偏差清单（P1/P2/P3）。
3. 最终可交付验收报告。
