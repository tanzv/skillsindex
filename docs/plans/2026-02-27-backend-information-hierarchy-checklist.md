# 后端统一信息层级规范与模块改版清单

## 1. 层级规范（统一规则）

### 1.1 视觉层级
1. L1：页面标题 + 核心 KPI/状态（首屏唯一最高权重）。
2. L2：主任务列表/主配置表单（用户主要操作区）。
3. L3：右侧上下文（说明、建议、快捷动作、日志流）。
4. L4：辅助提示（次级说明、路径、技术标注）。

### 1.2 配色规则
1. 高饱和蓝仅用于 CTA、当前选中、关键跳转。
2. 大面积容器使用低饱和底色，语义色用于小面积状态标签。
3. 亮色主题卡片与画布必须形成可感知分层。

### 1.3 布局规则
1. 1360 内容区默认左重右轻，右列承担上下文而非主任务。
2. 列表页保持“筛选 -> 列表 -> 详情/操作”阅读路径。
3. 同类页面采用统一骨架，但在标题区与关键卡片上区分页面类型。

## 2. 模块级检查清单

### 2.1 导入中心族（Manual/Zip/Repo/SkillMP）
- [ ] 导入入口卡在左上首位。
- [ ] 文件/仓库参数区与执行日志分块清晰。
- [ ] 右侧仅保留“当前导入上下文 + 风险提示 + 操作按钮”。
- [ ] 历史记录区弱化视觉噪声。

### 2.2 同步治理族（策略/台账/作业中心）
- [ ] 顶部明确页面类型（策略/台账/编排）。
- [ ] 主列表保留时序与状态为第一信息。
- [ ] 右侧动作区弱化背景饱和度，避免压过主列表。
- [ ] 运行事件流单独成块，保证扫描节奏。

### 2.3 账号与角色治理
- [ ] 列表页优先展示状态、角色、最近活动。
- [ ] 配置页表单区与策略区分层明确。
- [ ] 管理员危险动作视觉隔离但不过度抢眼。

### 2.4 身份与认证（SSO/DingTalk）
- [ ] Provider 列表、映射策略、回调日志三段式结构。
- [ ] 右侧保留健康状态与关键操作，不堆叠文本。
- [ ] 危险/高风险操作与普通操作分组。

### 2.5 审核治理（Case/Response/Postmortem）
- [ ] 案件摘要与处理动作优先。
- [ ] RCA/复盘内容保持低干扰阅读样式。
- [ ] 右侧行动建议只保留必要项。

### 2.6 运维合规（API Key/Ops/Release/Backup）
- [ ] 指标区和清单区信息密度平衡。
- [ ] 大面积说明块控制在 3-5 条要点。
- [ ] 关键开关（发布门禁、备份演练）具备明确状态对比。

## 3. 本次执行优先级

1. P1：右列抢焦点、亮色可读性不足、同步相关页面辨识弱。
2. P2：按钮与状态体系进一步收敛。
3. P3：跨模块固定导航锚点与二级信息重排。

## 4. 验收检查

1. 1 秒内定位页面主操作区。
2. 3 秒内完成主要状态扫描。
3. 10 秒内可区分同步策略/同步台账/异步编排三类页面。
4. 暗色与亮色切换后层级关系保持一致。
