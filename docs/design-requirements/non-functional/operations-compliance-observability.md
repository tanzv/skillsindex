# 运维、合规与可观测实施要求（目标态）

## 1. 目标

将非功能要求从“原则层”细化为“可执行治理基线”，用于发布门禁与长期运维。

## 2. 可观测性基线

### 2.1 指标体系

必须采集：

1. 请求量（QPS）
2. 延迟（P50/P95/P99）
3. 错误率（4xx/5xx）
4. 导入与同步任务成功率
5. 审计写入失败率

### 2.2 日志规范

关键日志字段：

1. `timestamp`
2. `level`
3. `request_id`
4. `user_id`（可空）
5. `route`
6. `error_code`
7. `latency_ms`

## 3. 审计留存与合规

1. 审计日志默认留存不少于 180 天
2. 支持按时间窗口导出（CSV/JSON）
3. 禁止业务层直接修改历史审计记录
4. 敏感字段（token、password）必须脱敏

## 4. 备份与恢复

1. 数据库每日全量 + 小时级增量备份
2. 每月至少一次恢复演练
3. 恢复目标：RPO <= 1 小时，RTO <= 4 小时（建议值）

## 5. 安全运维

1. 所有生产密钥通过密钥管理系统注入
2. API key / OAuth token 不允许写入明文日志
3. 高危操作启用二次确认与操作审计

## 6. 发布门禁

发布前必须满足：

1. 核心路径压测报告更新
2. 告警规则已部署并验证
3. 备份恢复演练在有效期内
4. 审计链路抽检通过

## 7. 验收标准

1. 发生故障时可通过 request_id 快速定位链路
2. 安全审计可追溯到关键治理动作
3. 备份可在目标时间内恢复
4. 非功能门禁项可重复执行且可审计
