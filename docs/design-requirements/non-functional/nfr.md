# 非功能需求

## 1. 性能（NFR-001~004）

### NFR-001 查询性能

- 市场关键词搜索 P95 <= 2 秒（1 万技能基线）

### NFR-002 页面体验

- 市场、分类、详情、后台核心页首屏 <= 3 秒（基线网络）

### NFR-003 写入性能

- 收藏/评分/评论/API Key 写操作 P95 <= 500ms

### NFR-004 外部调用超时

1. SkillMP 请求超时 20 秒
2. 钉钉请求默认超时 10 秒

### NFR-005 性能测量规范

性能验收必须基于统一口径执行：

1. 数据规模：至少 10,000 条技能记录
2. 并发基线：20~50 并发用户（按环境能力选择并固定）
3. 压测时长：预热 3 分钟 + 采样 10 分钟
4. 指标口径：P50/P95/错误率/吞吐量必须同时输出
5. 测试工具与脚本版本需记录在测试报告中

## 2. 可靠性（NFR-006~008）

### NFR-006 同步鲁棒性

- 仓库与 SkillMP 同步失败不覆盖旧数据

### NFR-007 事务一致性

- 技能内容与标签替换必须在同一事务提交

### NFR-008 失败可追踪

- 所有关键失败需记录日志并返回可定位错误信息

## 3. 安全（NFR-009~012）

### NFR-009 会话安全

- Cookie 必须签名，生产建议启用 `SESSION_COOKIE_SECURE=true`

### NFR-010 CSRF 防护

- 对所有非安全方法执行 CSRF token 校验

### NFR-011 API 鉴权

- 公开 API 必须携带有效 API Key
- 失效 key（撤销/过期）必须拒绝

### NFR-012 输入与路径安全

1. Zip 解压防路径逃逸
2. 仓库子路径防路径逃逸
3. 评论长度、评分范围等输入约束必须生效

## 4. 可观测性（NFR-013~015）

### NFR-013 日志追溯

审计日志、应用日志共同支持事后追溯。

### NFR-014 请求级关联

建议所有关键日志携带 request id，以支持跨层排障。

### NFR-015 指标与告警

建议补充指标采集（QPS、P95、错误率）与阈值告警策略。
