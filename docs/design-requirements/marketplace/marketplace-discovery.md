# 市场与发现需求

## 1. 功能总览

覆盖站点公开发现链路：

1. 市场主页 `/`
2. 分类总览 `/categories`
3. 分类详情 `/categories/{categorySlug}`
4. 时间线 `/timeline`
5. 技能详情 `/skills/{skillID}`

## 2. 功能需求

### FR-MKT-001 市场主页检索

- 支持检索参数：`q`、`tags`、`sort`、`category`、`subcategory`、`mode`
- `mode=keyword` 时使用结构化搜索
- `mode=ai` 时使用语义检索

规则：

1. 关键词模式分页大小固定为 24
2. 排序支持 `recent|stars|quality`
3. 仅返回公开技能（`visibility=public`）

验收标准：

1. 参数组合查询结果与过滤条件一致
2. 关键词模式支持前后页跳转
3. AI 模式在同条件下可返回语义相关结果

### FR-MKT-002 AI 语义检索行为

- 输入为空时返回空结果
- 结果上限受 `limit` 控制，页面场景默认 48
- 排序逻辑：语义分数优先，其次 star，再次更新时间

验收标准：

1. 对“自然语言问题”查询可返回相关技能
2. 同分技能时 star 更高优先

### FR-MKT-003 分类总览

- 展示每个一级分类的公开技能数量
- 展示子分类数量与描述信息

验收标准：

1. 分类计数与公开技能真实数量一致
2. 点击分类卡可进入分类详情

### FR-MKT-004 分类详情检索

- 分类页内支持二级筛选：`subcategory`、`q`、`tags`、`sort`
- 分类 slug 不存在时返回 404

验收标准：

1. 无效分类路径应返回 404
2. 分类页查询不应泄露非公开技能

### FR-MKT-005 时间线

- 支持 `interval=day|week|month`
- 展示每桶新增量与累计量（仅公开技能）

验收标准：

1. 默认 interval 为 week
2. 累计值单调不减

### FR-MKT-006 技能详情可见性

- 未登录仅可查看公开技能详情
- 已登录可查看公开技能 + 本人私有技能

验收标准：

1. 未授权访问私有技能返回 404
2. 管理员不自动越权浏览他人私有技能详情（保持当前实现口径）

### FR-MKT-007 详情页互动聚合展示

详情页需聚合：

1. 收藏数
2. 评分人数与均分
3. 评论列表（默认上限 80）
4. 当前用户是否已收藏与个人评分

### FR-MKT-008 本地化入口

- 支持 `/zh` 与 `/zh/*` 语言别名路由
- 支持通过 `lang=en|zh` 切换
- 支持 `/skillsmp` 兼容别名入口（重定向到市场首页）

## 3. 页面级非功能要求

1. 市场搜索 P95 <= 2 秒（1 万技能基线）
2. 分类页与详情页首屏渲染 <= 3 秒（基线环境）
3. 错误页面应返回可诊断信息，不暴露内部堆栈
