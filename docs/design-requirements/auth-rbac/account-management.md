# 账号管理需求（最终完整性目标）

## 1. 目的与范围

本文件定义“最终 Web 完整形态”中的账号管理能力，覆盖：

1. 用户端账号中心
2. 管理端账号治理
3. 安全与审计约束

说明：本文件属于目标态需求设计，部分能力当前代码尚未实现。

## 2. 用户端账号管理

### FR-ACC-001 账号中心入口

系统应提供用户账号中心入口（如 `/account`），至少包含：

1. 基础资料页
2. 安全设置页
3. 会话管理页

### FR-ACC-002 基础资料管理

用户可查看并修改基础资料：

1. 昵称/显示名
2. 头像（可选）
3. 个人简介（可选）

验收标准：

1. 非法输入（超长、非法字符）被拦截
2. 成功更新后详情页可立即读取到新值

### FR-ACC-003 密码修改

登录用户可在安全设置页修改密码：

1. 必须输入当前密码
2. 新密码满足复杂度策略（至少长度要求）
3. 修改成功后可配置“踢出其他会话”

### FR-ACC-004 忘记密码与重置

系统应支持忘记密码流程：

1. 发送重置凭证（邮件或短信，按部署能力选型）
2. 凭证具备时效与一次性约束
3. 重置成功后旧凭证失效

### FR-ACC-005 会话管理

用户可查看活跃会话列表并撤销指定会话，至少展示：

1. 登录时间
2. 最近活动时间
3. 设备/客户端信息（可选）

### FR-ACC-006 账号停用（可选能力）

用户可发起账号停用请求，系统需二次确认并保留可恢复策略说明。

## 3. 管理端账号治理

### FR-ACC-007 用户列表与检索

管理端应提供用户列表，支持：

1. 用户名检索
2. 角色过滤
3. 状态过滤（active/disabled/locked）

### FR-ACC-008 账号状态治理

`admin/super_admin` 可执行：

1. 禁用账号
2. 启用账号
3. 强制退出全部会话

权限规则：

1. 普通 admin 不得操作 super_admin 账号状态
2. 关键操作需二次确认

### FR-ACC-009 管理员密码重置

super_admin 可发起“重置用户密码”流程，必须满足：

1. 不直接展示明文密码
2. 通过一次性重置链接或临时凭证完成
3. 重置动作写入审计日志

### FR-ACC-010 角色治理一致性

角色更新必须与既有规则一致：

1. 禁止降级最后一个 super_admin
2. 所有角色变更可追溯

## 4. 安全与审计要求

### FR-ACC-011 安全控制

1. 高频失败登录应触发限流/临时锁定策略
2. 敏感操作需 CSRF 与会话双重校验
3. 重置凭证需哈希存储或安全签名

### FR-ACC-012 审计覆盖

以下动作必须审计：

1. 资料更新
2. 密码修改/重置
3. 会话撤销
4. 账号禁用/启用
5. 角色变更
