# 首页原型配色规范（黑白主色 + 辅助色）

## 1. 目标

本规范用于统一用户端首页原型（Marketplace Command Deck 家族）的配色语言，确保：

1. 主视觉以黑白灰为核心，不再使用蓝色作为主色。
2. 标签（Tag）与卡片高亮（Card Highlight）遵循一致规则。
3. 辅助色仅用于状态语义，不用于主结构与大面积背景。

## 2. 适用范围

- 原型文件：`prototypes/skillsindex_framework/skillsindex_framework.pen`
- 首页家族页面（含断点变体）：
  - `j0pbU`（Desktop Dark）
  - `EbJ9a`（Desktop Light）
  - `bCPvI`（Tablet Light）
  - `Dp0rP`（Mobile Light）

## 3. 基础色板

### 3.1 主色（黑白灰）

| Token | 用途 | Hex |
|---|---|---|
| `bg.canvas` | 页面底色 | `#0A0A0B` |
| `bg.surface.1` | 顶栏/一级容器 | `#111111` |
| `bg.surface.2` | 二级容器/分区底 | `#1C1D22` |
| `bg.surface.3` | Chip/Tag 底色 | `#2A2A2D` |
| `line.subtle` | 分割线/轻边框 | `#52525B` |
| `text.primary` | 主文字 | `#F1F1F1` |
| `text.secondary` | 次级文字 | `#D4D4D8` |
| `text.tertiary` | 弱化文字 | `#B3B3B8` |

### 3.2 辅助色（仅状态语义）

| Token | 用途 | Hex |
|---|---|---|
| `state.success.bg` | 成功状态底色 | `#14532D` |
| `state.success.fg` | 成功状态文字 | `#BBF7D0` |
| `state.warning.bg` | 警告状态底色 | `#7C2D12` |
| `state.warning.fg` | 警告状态文字 | `#FED7AA` |
| `state.danger.bg` | 危险状态底色 | `#7F1D1D` |
| `state.danger.fg` | 危险状态文字 | `#FECACA` |

## 4. Tag 标签规范

### 4.1 默认标签

- 背景：`#2A2A2D`
- 文字：`#E5E5E7`
- 圆角：保持现有药丸或圆角方胶囊风格

### 4.2 选中标签

- 背景：`#111111`
- 文字：`#F1F1F1`
- 可选描边：`#52525B`（用于浅色页增强边界）

### 4.3 状态标签

- 仅允许成功/警告/危险语义使用辅助色。
- 禁止将辅助色用于“分类标签”“筛选标签”“普通信息标签”。

## 5. 卡片高亮规范

### 5.1 卡片层级

- 一级高亮卡：`#1C1D22`
- 二级高亮卡：`#111111`
- 普通卡：`#111111` 或 `#FFFFFF`（仅浅色页面）
- 高亮差异通过层级灰度体现，不使用蓝底卡片。

### 5.2 卡片内文案

- 标题：`text.primary`
- 描述：`text.secondary`
- 元信息/次要说明：`text.tertiary`
- 操作文本（Details/Install/Queue）：默认 `#D4D4D8`，激活 `#F1F1F1`

## 6. Hover 与动画规范

### 6.1 基础原则

- Hover 反馈必须可感知，但不能抢夺主要信息层级。
- 仅允许使用 `background`、`textColor`、`stroke`、`opacity`、`translateY`、`shadow` 做过渡。
- 禁止因为 Hover 导致布局抖动（如高度变化、文本换行、容器重排）。

### 6.2 时长与缓动

- Hover-in：`120ms`，`ease-out`
- Hover-out：`100ms`，`ease-in`
- Press/Active：`80ms`，`ease-out`
- 页面级入场动效（可选）：`160ms` ~ `220ms`，仅用于首次进入，不循环

### 6.3 Tag 标签 Hover

- 默认：背景 `#2A2A2D`，文字 `#E5E5E7`
- Hover：背景 `#343438`，文字 `#F1F1F1`，描边 `#52525B`
- 选中态 Hover：保持选中语义，禁止切换到辅助状态色

### 6.4 卡片 Hover（包含 Skill 卡片与精选卡片）

- 背景：在当前层级基础上上浮一档（例如 `#111111 -> #1C1D22`）
- 位移：`translateY(-2px)`
- 阴影：`0 10 24 rgba(0,0,0,0.25)`
- 标题/操作文本：从 `text.secondary` 提升到 `text.primary`

### 6.5 按钮与操作项 Hover

- 中性按钮：`#2A2A2D -> #343438`
- 主操作按钮（黑底）：`#111111 -> #1C1D22`
- 文本颜色：`#D4D4D8 -> #F1F1F1`
- 禁止使用蓝色 Hover 高亮替代中性层级变化

### 6.6 禁止项

- 禁止弹跳、抖动、旋转等装饰性动效
- 禁止单次交互动画总时长超过 `240ms`
- 禁止非加载场景下的无限循环动画
- 禁止 Hover 闪烁或高频明暗切换

### 6.7 无障碍建议

- 如果运行环境启用 reduced motion，禁用位移与阴影动画，仅保留颜色过渡
- Hover 与 Focus-visible 的语义必须一致，避免键盘用户与鼠标用户反馈不一致

## 7. 禁用色清单（首页范围）

以下色值不得在首页的结构层、标签、卡片高亮、操作文本中继续使用：

- `#1E3A8A` `#183464` `#274B8E` `#315CA7`
- `#2563EB` `#1D4ED8` `#3B82F6` `#60A5FA`
- `#93B4E8` `#8FB4EA` `#A8CBFF` `#CDE1FF` `#BFD2ED` `#DBEAFE`
- `#0B1326` `#0E1B30` `#0F172A` `#1F2937`
- `#112447` `#0E1A31` `#081223`

## 8. 审查与验收

### 7.1 审查项

1. Tag 标签：背景、文字、选中态是否符合第 4 章。
2. 卡片高亮：是否仅使用黑白灰层级，不含蓝底高亮。
3. 操作文案：链接/按钮文本是否已去蓝化。
4. 断点一致性：Desktop/Tablet/Mobile 同一模块语义颜色一致。

### 7.2 通过标准

1. 首页家族页面中，禁用色清单在上述目标元素中的命中数为 0。
2. Tag 与卡片在深浅页面具备一致的语义映射。
3. 状态色仅出现在状态语义，不出现在结构性背景。

## 9. 变更控制

后续若需要新增颜色，必须满足：

1. 先定义语义 Token，再落到具体 Hex。
2. 不得新增“品牌蓝”作为主结构色。
3. 新增颜色必须同步更新本规范与审查脚本。
